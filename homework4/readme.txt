1.使用說明

	1-1. 本系統為一個簡易的回合制卡牌對戰遊戲，以下為遊戲流程的簡述：

		a. 載入玩家資料與戰場配置

		b. 每回合進行卡片的召喚佈署(佈局階段)

		c. 依照技能速度 Action Speed 進行技能階段(技能階段)

		d. 依序進行攻擊階段(攻擊階段)

		e. 回合數來到 10 回合，或直到一方生命歸零時會結束遊戲並宣布勝負

		f, 每個回合結束時選擇是否存檔

		註1. 所有玩家生命值、手牌卡片數量、場上卡片會更新到player_info.txt 與 playground.txt

		註2. 若遊戲結束後選擇存檔，程式會自動更新所有資料後退出。若選擇不存檔則直接進入下一回合，遊戲並不會在玩家選擇繼續遊戲自動保存每個回合，還請玩家特別留意。

		註3. 欲查閱遊戲規則請參考readme.txt說明第3點



	1-2. 以下為使用本系統的方法

		a. 在收放hw4的資料夾打開cmd，並在cmd裏頭編譯hw4
		ex. javac hw4.java

		b. 執行hw4, 還請使用者另外輸入兩個參數player_info.txt playground.txt(儲存檔案)
		ex. hw4 player_info.txt playground.txt

		c. 程式將自動讀取存檔後開始並顯示遊戲畫面與指令說明，請玩家依照提示輸入指令

		d. 程式會依指令結果自動進入技能階段、戰鬥階段，並更新場地



2.使用上的須知

3.遊戲規則與流程

	a. 遊戲目標：
	玩家需在 10 回合內將對手生命歸零；若超過 10 回合仍未分出勝負，則比較生命值高低決定勝負。若生命相同，則為平手。
		- 任一玩家生命歸零 → 另一方獲勝
		- 雙方在同一回合同時死亡 → 平手
		- 回合數到達 10 → 比較雙方生命決定勝負
		- 若生命相同 → 平手			
	


	b. 玩家資訊：
		每位玩家擁有：
			- 生命值
			- 能量
			- 手牌(最多三張)

		註. 回合開始時，玩家的能源值會自動設定為當前回和數
	


	c. 卡片種類
		遊戲共有四種卡片，每種卡片擁有不同生命、攻擊力與技能：

		priest
		   - Life: 200
		   - ATK: 50
		   - 召喚能量消耗：2
		   - 技能：回復己方全體生命

		tank
		   - Life: 300
		   - ATK: 60
		   - 召喚能量消耗：2
		   - 技能：回復自身生命

		shooter
		   - Life: 150
		   - ATK: 100
		   - 召喚能量消耗：3
		   - 技能：對敵軍全體造成10點固定傷害

		assassin
		   - Life: 150
		   - ATK: 70
		   - 召喚能量消耗：3
		   - 技能：直接對敵方玩家造成50點固定傷害

		
		在技能階段時，卡片技能發動優先順序為 priest --> tank --> shooter  --> assassin



	d. 回合流程

	每回合包含三個主要階段：

	
		階段 1：佈局階段
			玩家會在此階段可以進行卡片召喚、顯示資料與結束回合的動作，此三項動作皆以指令方式使用，佈局玩家行動順序：P1 → P2


			- put [class] [position]
			   將一張卡片召喚到指定位置(1~3)

			- show game status
			   顯示玩家資訊與戰場狀態

			- end layout stage
			   結束本階段，進入技能階段
		
			註.召喚限制：
			- 玩家需持有該類卡片
			- 玩家能量須足夠
			- 位置不得被佔用

		階段 2：技能階段(Skill Phase)
			- 技能階段由系統自動執行，不需玩家輸入，技能玩家行動順序：P1 → P2
			- 技能執行由priest --> tank --> shooter  --> assassin排序來發動技能，若有卡片死亡會直接移除該卡
		
		階段 3：戰鬥階段(Battle Phase)
			- 技能階段結束後開始攻擊階段，佈局玩家行動順序：P1 → P2

		
			攻擊規則：
			- 若雙方在同一位置皆有卡牌 → 互相攻擊
			- 若敵方位置為空 → 直接攻擊玩家，扣除生命值

			每次攻擊後，如卡片生命為0，則移除該卡。



	e. 存檔說明
	每回合結束後系統會詢問是否儲存進度：

		輸入 Y：
		- 更新 player_info.txt
		- 更新 playground.txt
		- 結束遊戲

		輸入 N：
		- 進入下一回合



4.以下為程式指令範例

	a. put [class name] [position]
	佈署手上的單位至指定位置
	ex. put assassin 1
	    put tank 2

	b. show game status
	顯示當前的場地與戰鬥資料，包含：回合數玩家生命、能量，玩家手牌擁有卡片列表與戰場三格區域卡片資訊
	ex. show game status

	c. end layout stage
	結束自己的佈署階段，進入下個階段
	ex. show individual score Alice

	d.Y / N (在存檔階段使用)
	選擇是否要存檔，Y為存檔，N為繼續遊戲，此指令在佈局階段不可使用
	ex. Y
